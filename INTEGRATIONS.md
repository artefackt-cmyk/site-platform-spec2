# Integrations

## Общий принцип

Каждый внешний сервис подключается через единый контракт и отдельный adapter. Бизнес-логика платформы не должна зависеть от специфики конкретного провайдера.

## Первая очередь

### МойСклад

Функции:

- импорт товаров;
- импорт вариантов;
- импорт категорий;
- импорт цен;
- импорт остатков;
- импорт изображений при поддерживаемом сценарии;
- обновление товаров;
- передача заказов;
- передача покупателей;
- журнал синхронизации;
- повтор ошибок;
- ручной запуск синхронизации;
- настройка владельца каждого поля.

### ЮKassa

- создание платежа;
- проверка статуса;
- webhooks;
- возвраты;
- идемпотентность;
- СБП через поддерживаемый сценарий;
- передача данных для онлайн-кассы, когда применимо.

### СДЭК

- расчет доставки;
- пункты выдачи;
- курьерская доставка;
- создание отправления на следующем этапе;
- статусы доставки;
- обработка ошибок API.

### Яндекс Метрика

- счетчик;
- ecommerce-события;
- цели;
- настройка через интерфейс проекта.

### Telegram

- уведомления о заказах;
- уведомления о заявках;
- ошибки интеграций;
- важные системные события.

### Email

- подтверждение заказа;
- изменение статуса;
- ссылка на цифровой продукт;
- заявки из форм;
- технические уведомления.

## Вторая очередь

- amoCRM;

## Отраслевые интеграции

- Posiflora;
- специализированные CRM и учетные системы.

## Требования ко всем интеграциям

- безопасное хранение секретов;
- шифрование чувствительных данных;
- webhooks с проверкой подписи;
- идемпотентность;
- retry policy;
- dead-letter обработка;
- журнал событий;
- health status;
- понятные ошибки для пользователя;
- возможность отключить интеграцию без повреждения данных.
