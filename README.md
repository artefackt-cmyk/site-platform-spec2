# Стартовый пакет спецификации платформы

Этот архив содержит начальную продуктовую документацию для создания репозитория и последующей разработки через Codex.

## Файлы

- PRODUCT_VISION.md — концепция и позиционирование;
- MVP_SCOPE.md — границы первой версии;
- USER_ROLES.md — роли пользователей;
- USER_JOURNEYS.md — ключевые сценарии;
- DOMAIN_MODEL.md — доменные сущности;
- INTEGRATIONS.md — очередность и правила интеграций;
- ROADMAP.md — этапы разработки;
- AGENTS.md — инструкции для Codex.

## Следующий шаг

1. Создать пустой репозиторий GitHub.
2. Распаковать файлы в корень репозитория.
3. Подключить Codex к репозиторию.
4. Использовать отдельную задачу для инициализации monorepo.

Не просите Codex сразу реализовать всю платформу. Первая задача должна ограничиваться техническим каркасом проекта.
